    kandinsky

    Visual rendering of sound in real-time 3D

    Components

    The main components of kandinsky are:

    • The audio processing server, running on the hardware that records the audio input. The server application performs signal transformation and feature extraction, such as beat detection. Results from feature extraction are streamed to clients. A setup of kandinsky may have several servers running that specialize in different tasks.
    • Clients that interpret data received from a kandinsky server, usually as a 3D rendering.

    The processing on both server- and client side consists of daisy-chained plugins that are configured in the kandisky main application.

    Communication protocols

    If GUI events are provided by a server instance, clients can subscribe to its GUI socket. GUI Messages are broadcasted as JSON objects.

    Real-time data from feature extraction is sent on one-way channels using simple binary packages broadcasted via UDP.

    Dependencies

    The kandinsky project is built depending on the following libraries:

    Server application

    • EMBB, optionally using my modified fork of EMBB
    • FFTW-EMBB

    Client application

    • GTK+ for clients with GUI dialogs
    • OpenGL and the official or modified version of EMBB for 3D rendering clients

    Build instructions

    Clone this repository and use CMake to generate a native build environment.
